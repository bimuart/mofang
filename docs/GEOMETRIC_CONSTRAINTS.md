# 3×3 魔方 — 几何约束与可解性（面向实现与 AI 阅读）

## 1. 本文档的目的

| 项目 | 说明 |
|------|------|
| **读者** | 第一次接触本项目的 AI Agent、新加入的开发者。 |
| **目标** | 用**最少前置知识**，说明：3×3 魔方在数学上如何建模、**哪些条件是「合法且可转动还原」的必要条件**、以及实现填色校验与求解时要注意什么。 |
| **不重复的内容** | 具体 UI、步骤记号列表、求解算法伪代码；这些见需求文档与实现代码。 |

**关键结论（先读）**：一个 3×3 魔方状态 **能够仅通过面转动还原**，当且仅当（在标准建模下）同时满足下面 **四类约束**。前两类常被称作 **朝向（orientation）约束**，第三类是 **排列奇偶**；第四类是 **每块贴纸 multiset（块身份）**。四类缺一不可作为「可解」的完整判据。

---

## 2. 背景：魔方在数学上是什么

### 2.1 物理对象

- 3×3×3 魔方由 **26 个** 外露的小立方体（**cubie**）组成：**6 个中心块、12 个棱块、8 个角块**（内部还有不可见的机械核心，建模时通常忽略）。
- **中心块** 在经典 3×3 上 **相对彼此不会互换位置**（只能随整体旋转魔方而转动朝向；若仅考虑六色标准魔方，中心块无「图案朝向」问题）。因此 **六个面的颜色由六个中心块唯一确定**。
- **棱块** 始终在棱的位置上移动；**角块** 始终在角的位置上移动。棱与角 **不会互换角色**。

### 2.2 「状态」包含什么

完整描述一个可转动魔方状态，通常需要：

1. **哪些棱块在哪个棱槽位、是否翻转**；
2. **哪些角块在哪个角槽位、相对该槽位扭转了多少**；
3. **六个中心** 相对于空间固定坐标轴的朝向（标准六色魔方常简化为「已由用户固定」）。

用户自定义「每块每贴纸是什么颜色」时，本质上是在约束上述信息；**并非任意贴纸分配都对应真实可转动到达的状态**。

### 2.3 为什么需要「约束」而不是随便填色

若把魔方 **拆开再随机装回**，可能得到 **无法通过任何合法转动序列还原** 的状态。可解性理论给出 **充要条件**（在固定中心与标准块类型的前提下）：下面四条全部满足 ⟺ 存在合法转动序列回到还原态。

因此：做 **自动填色推理**、**合法性检查**、或 **调用求解器之前**，都应保证（或检测）这些约束。

## 3. 记号与约定（实现时必须统一）

### 3.1 面与颜色

需求中 **六个面采用 Singmaster 面记号**，与转动字母一致：**U、D、L、R、F、B**，分别对应 **上、下、左、右、前、后**（Up / Down / Left / Right / Front / Back）。六种颜色与六个面 **一一对应**；几何上与空间固定坐标轴绑定，**持有魔方者正对的那一面为 F** 等约定须在实现中写死，保证 3D、填色与步骤语义一致。

**俯视水平环（与 [REQUIREMENTS.md §2.4](./REQUIREMENTS.md#24-已确认的产品与交互约定实现须遵守) 一致）**：自 **U 方向向下俯视**（从上往下看），四个侧面沿 **顺时针** 依次为 **F → L → B → R**（闭环）。Three.js 等实现中，各外法线、面片命名与该环一致，可避免「步骤正确但箭头/转向观感反了」。**Singmaster 中单步转动的「顺时针」** 仍定义为 **面向该面正视**（法线指向观察者）时的顺时针，与上述俯视环独立、勿混用。

### 3.2 槽位（slot）与块（piece）

- **槽位**：魔方上固定的 12 个棱位置、8 个角位置（在空间中命名，如 UFR 角等）。
- **块**：物理上的 12 个棱块、8 个角块；每一块在还原态下对应 **一组贴纸颜色**（棱为二元组，角为三元组），称为该块的 **身份（identity）**。

用户自定义时，是在把「某块」放到「某槽」并指定各面贴纸颜色；**可解性**要求：全局上存在一种 **仅通过旋转** 可达的赋值，与这些约束一致。

### 3.3 还原态（reference）

所有 **扭转量、翻转、置换** 都相对某一 **约定的还原态** 定义：

- 还原态由 **用户确认的六面中心颜色** 决定：每个面纯色，且与对应中心一致。
- 一旦选定，**每个棱槽、角槽** 在还原态下应贴哪些颜色是确定的，从而每块在还原态下应处在哪个槽也可推导（在颜色两两区分的前提下）。

实现中必须 **固定一套「槽位命名 + 还原态贴纸布局」**，否则「扭转」「翻转」「置换」无法数值化。

## 4. 四类几何（代数）约束

以下陈述在 **标准 3×3、六色、每色一面、块不替换** 的模型下成立。

---

### 约束 A — 块身份 / 贴纸多重集（色位守恒）

**陈述**：  
- 每个 **棱块** 暴露的两种颜色（不计顺序）在整个转动过程中 **不变**；  
- 每个 **角块** 暴露的三种颜色（不计顺序） **不变**；  
- 中心块各对应一种颜色。

**推论**：  
- 不可能出现「角块上有两种红色」这类与物理块类型矛盾的组合；  
- 用户填色时，每一块所填颜色集合必须与 **该块类型在还原态下应有的颜色集合** 一致（在中心配色确定后，通常每块身份唯一确定）。

**实现提示**：  
这是 **局部约束**，也是定义「哪块是哪个」的基础。若中心未对齐或颜色重复，需单独定义规则（本项目以 **U、D、L、R、F、B** 六面区分为准）。

---

### 约束 B — 角块扭转（corner orientation / twist）

**陈述**：  
在 **一致定义** 的角块朝向下，设 8 个角块各有一个扭转值 \( t_i \in \{0,1,2\} \)（表示相对槽位参考方向转了多少个 \(120^\circ\)），则：

\[
\sum_{i=1}^{8} t_i \equiv 0 \pmod{3}.
\]

**直观**：角块只能三种朝向循环；全局八种扭转不能「孤立地」只改一两个而不顾整体，**总和必须 mod 3 为 0**。

**实现提示**：  
- 必须先定义 **每个角槽位** 的「0 朝向」参考（通常与 U/D 面法线相关）；  
- 不同文献定义会使 \( t_i \) 数值不同，但 **mod 3 的和** 作为可解条件不变。

---

### 约束 C — 棱块翻转（edge orientation / flip）

**陈述**：  
在 **一致定义** 的棱块朝向下，每条棱块要么是「未翻转」、要么是「翻转」；**翻转棱的条数为偶数**。

**直观**：单条棱不能单独保持其它块不变而只翻一条；全局翻转奇偶受偶性约束。

**实现提示**：  
与角块类似，需为每条棱定义 **0/1 翻转** 规则（常参考 U/D 或四轴边）；数值依赖定义，**奇偶性** 为不变量。

---

### 约束 D — 排列奇偶（permutation parity）

**陈述**：  
设相对还原态：

- 角块产生一个置换 \( \sigma_c \in S_8 \)（8 个角块在 8 个角槽上的排列）；  
- 棱块产生一个置换 \( \sigma_e \in S_{12} \)（12 个棱块在 12 个棱槽上的排列）。

可解的必要条件为：

\[
\operatorname{sgn}(\sigma_c) = \operatorname{sgn}(\sigma_e),
\]

即 **角置换与棱置换同为偶置换或同为奇置换**。

**直观**：  
合法转动生成的角块置换与棱块置换，其 **奇偶性始终联动**。仅通过「交换两个角」而不调整棱（或反之）可能违反该关系，从而 **不可解**。

**为何必须单独列出**：  
仅验证 **约束 B + 约束 C**（角扭转和与棱翻转偶性）**不足以** 保证可解；存在满足 B、C 但仍不可解的状态，典型来源是 **拆块重装** 或随机赋值导致的 **置换奇偶不一致**。

---

## 5. 四条约束之间的关系与「完整性」

在经典结论中，对于 **3×3 魔方合法转动可达的状态集合**：

- **约束 A** 限定每块贴纸 multiset；  
- 在 A 成立且中心固定的前提下，**B + C + D** 共同刻画 **可解性**（与「从还原态经合法转动可达」等价）。

因此，产品需求中的 **「基本几何逻辑」** 应 **明确包含 D（排列奇偶）**，与 **A、B、C** 并列。

## 6. 与「求解器」的关系

| 话题 | 说明 |
|------|------|
| **约束检验** | 在运行 Kociemba、两阶段法或其它求解器前，应用 **A～D** 验证状态；不满足则不可解，应提示用户。 |
| **求解算法本身** | 属于 **搜索与群论算法**，不在本文展开；它不替代上述判据，而是在判据满足时 **找步骤**。 |
| **逆向步骤** | 若已有「定义态 → 还原态」的转动序列，其 **逆序列** 即为「还原态 → 定义态」；需与相同记号约定。 |

## 7. 实现清单（供 AI / 开发者自检）

- [ ] 固定 **面记号 U～B ↔ 空间方向** 与 **还原态** 布局。  
- [ ] 为 **8 角**、**12 棱** 定义槽位名称与 **角扭转 / 棱翻转** 的 0 参考。  
- [ ] 由用户输入计算：每块身份是否满足 **约束 A**。  
- [ ] 计算 **8 个角扭转** 之和 mod 3 是否为 0 — **约束 B**。  
- [ ] 计算 **翻转棱数** 是否为偶 — **约束 C**。  
- [ ] 计算角、棱 **置换的符号**，检查是否相等 — **约束 D**。  
- [ ] 填色推理时：候选颜色缩减须 **不破坏 A～D 的可行性**（可能需 CSP / 回溯，仅贪心可能不够）。

## 8. 扩展说明（本项目可忽略除非需求变更）

- **图案中心 / Supercube**：中心块有朝向时，需额外约束。  
- **4×4 及以上**：存在 **中心块组**、**棱块不可区分** 等更复杂现象，与 3×3 不同。  
- **仅换色不改结构的「上帝数」等**：与可解性判据无关。

## 10. 与 UI「填色」的关系（实现注记）

几何上，**单个贴纸**在完整合法状态下所能出现的面记号，受所属 **棱块 / 角块** 的身份限制（例如某一角位上的三面贴纸，仅可能出现该角块三种颜色中的某一种）。若在交互式填色 **未完成** 时，仅用「块 multiset」去限制 **调色板可点颜色**，则会出现 **角位仅 3 色、棱位仅 2 色** 可点的情况，与「空白态下可先尝试六种面记号」的产品体验不一致。

因此实现上采用 **两层语义**（详见 [REQUIREMENTS.md §3.4.1](./REQUIREMENTS.md#341-填色扩展锁定三级候选冲突判定与调色板实现约定)）：

- **严格**：块 multiset + 可解性延拓 → **冲突格**、**推荐 / 普通候选**；
- **宽松（仅调色板）**：在**锁定格固定**的前提下，仅校验 **全局九色计数** → 空白态下一般 **六种均可点**；填色后若与严格几何不符，由 **冲突高亮** 提示用户修正。

四条不变量 **A～D** 的完整检验仍仅在 **可判定时**（如完全填色）用于求解与可解性结论。
